# Plan Skill

When the task is non-trivial:

1. Restate requirements and constraints.
2. Identify affected packages and files.
3. Provide phased steps with dependencies.
4. Call out risks and edge cases.
5. Include a test or verification plan.
6. Ask for confirmation before editing code.
