# Craft Agents 添加资源（Sources）完全指南

## 📁 资源存储位置

所有资源存储在工作区目录下：
```
~/.craft-agent/workspaces/{workspaceId}/sources/{sourceSlug}/
├── config.json      # 配置文件（必需）
├── guide.md         # 使用指南（推荐）
├── permissions.json # Explore模式权限（推荐）
└── icon.png/svg     # 图标（可选）
```

---

## 🔌 三种资源类型

### 1️⃣ MCP Sources（模型上下文协议服务器）

#### HTTP/SSE 远程服务器（OAuth 认证）
```json
{
  "type": "mcp",
  "name": "Linear",
  "slug": "linear",
  "provider": "linear",
  "enabled": true,
  "mcp": {
    "url": "https://mcp.linear.app",
    "authType": "oauth"
  }
}
```

#### Stdio 本地服务器（命令行运行）
```json
{
  "type": "mcp",
  "name": "GitHub",
  "slug": "github",
  "provider": "github",
  "enabled": true,
  "mcp": {
    "transport": "stdio",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-github"],
    "env": {
      "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx"
    },
    "authType": "none"
  }
}
```

#### 常见 MCP 服务器示例

| 服务 | 命令 | 环境变量 |
|------|------|---------|
| **文件系统** | `npx -y @modelcontextprotocol/server-filesystem /path` | 无 |
| **Brave 搜索** | `npx -y @modelcontextprotocol/server-brave-search` | `BRAVE_API_KEY` |
| **Memory** | `npx -y @modelcontextprotocol/server-memory` | 无 |
| **GitHub** | `npx -y @modelcontextprotocol/server-github` | `GITHUB_TOKEN` |

### 2️⃣ API Sources（REST API）

#### Bearer Token 认证
```json
{
  "type": "api",
  "name": "OpenAI",
  "slug": "openai",
  "provider": "openai",
  "enabled": true,
  "api": {
    "baseUrl": "https://api.openai.com/v1/",
    "authType": "bearer",
    "testEndpoint": {
      "method": "GET",
      "path": "models"
    }
  }
}
```

#### Header 认证（X-API-Key 风格）
```json
{
  "type": "api",
  "name": "Exa Search",
  "slug": "exa",
  "provider": "exa",
  "enabled": true,
  "api": {
    "baseUrl": "https://api.exa.ai/",
    "authType": "header",
    "headerName": "x-api-key",
    "testEndpoint": {
      "method": "POST",
      "path": "search",
      "body": { "query": "test", "numResults": 1 }
    }
  }
}
```

#### Basic 认证（用户名+密码）
```json
{
  "type": "api",
  "name": "Jira",
  "slug": "jira",
  "provider": "jira",
  "enabled": true,
  "api": {
    "baseUrl": "https://your-domain.atlassian.net/rest/api/3/",
    "authType": "basic",
    "testEndpoint": {
      "method": "GET",
      "path": "myself"
    }
  }
}
```

## 🔐 认证方式

创建资源后，根据 `authType` 触发不同的认证流程：

| 认证类型 | 触发工具 | 说明 |
|---------|---------|------|
| `oauth` (MCP) | `source_oauth_trigger` | 打开浏览器进行 OAuth |
| `oauth` (Google) | `source_google_oauth_trigger` | Gmail/Drive/Calendar |
| `oauth` (Slack) | `source_slack_oauth_trigger` | Slack 工作区 |
| `oauth` (Microsoft) | `source_microsoft_oauth_trigger` | Outlook/Teams |
| `bearer` | `source_credential_prompt` mode="bearer" | 输入 API Token |
| `header` | `source_credential_prompt` mode="header" | 输入 API Key |
| `basic` | `source_credential_prompt` mode="basic" | 输入用户名密码 |
| `none` | 无需认证 | 直接可用 |

## 📋 完整添加流程

### 步骤 1：创建资源文件夹和配置

```bash
# 创建资源目录
mkdir -p ~/.craft-agent/workspaces/{workspaceId}/sources/my-source

# 创建 config.json
```

### 步骤 2：编写 guide.md（推荐）

```markdown
# My Source

这个资源用于 [具体用途]。

## Scope
- 可访问的数据范围
- 相关项目/团队

## Guidelines
- 使用最佳实践
- 注意事项和限制
```

### 步骤 3：配置 permissions.json（Explore 模式）

```json
{
  "allowedMcpPatterns": [
    { "pattern": "list", "comment": "列表操作" },
    { "pattern": "get", "comment": "读取操作" },
    { "pattern": "search", "comment": "搜索操作" }
  ]
}
```

### 步骤 4：验证配置

使用 `source_test` 工具验证配置是否正确。

### 步骤 5：触发认证

根据认证类型调用相应的认证工具。
