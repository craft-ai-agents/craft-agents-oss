# Craft Agents vs Claude Code 深度对比分析

## 🎯 产品定位差异

| 维度 | Claude Code | Craft Agents |
|------|-------------|--------------|
| **核心定位** | CLI 编程助手 | 桌面文档中心化工作平台 |
| **目标用户** | 开发者（终端用户） | 知识工作者（更广泛） |
| **交互方式** | 终端命令行 | 图形界面 (Electron) |
| **使用场景** | 代码编写、调试、Git操作 | 多源数据整合、工作流自动化 |
| **哲学理念** | "Pair Programming" | "Document-Centric AI" |

## ⚡ 核心差异详解

### 1. **MCP 集成深度**

**Claude Code:**
- MCP 是**可选扩展**，不是核心
- 用户需要自己配置 `~/.claude/settings.json`
- 没有内置的认证管理
- 环境变量需要手动传递

**Craft Agents:**
- MCP 是**第一公民**，核心抽象
- 图形化的 Sources 管理界面
- **内置 OAuth 2.0 + PKCE** 认证流程
- 自动的凭证加密存储 (AES-256-GCM)
- 环境变量安全过滤（防止泄露敏感信息给子进程）

```typescript
// Craft Agents 的 MCP 安全隔离
const BLOCKED_ENV_VARS = [
  'ANTHROPIC_API_KEY',  // 永远不会泄露给 MCP 服务器
  'AWS_SECRET_ACCESS_KEY',
  'GITHUB_TOKEN',
  // ...
];
```

### 2. **权限系统对比**

| 特性 | Claude Code | Craft Agents |
|------|-------------|--------------|
| **三级模式** | ✅ Safe/Ask/Auto | ✅ Safe/Ask/Auto |
| **Bash 命令分析** | ✅ 内置 | ✅ 内置 |
| **工具级规则** | ❌ 无 | ✅ 每个工具可配置 |
| **源级规则** | ❌ 无 | ✅ 每个 Source 可配置 |
| **持久化规则** | ⚠️ 有限 | ✅ 完整配置文件 |
| **切换方式** | `--safe`/`--auto` 或 Shift+Tab | Shift+Tab 或 UI |

**Craft Agents 的细粒度权限示例:**
```json
{
  "permissions": {
    "defaultMode": "ask",
    "toolRules": {
      "Bash": "ask",
      "Read": "allow-all",
      "Write": "ask"
    },
    "sourceRules": {
      "github": "allow-all",
      "production-db": "safe"  // 生产数据库强制只读
    }
  }
}
```

### 3. **会话管理**

**Claude Code:**
- 会话存储在 `~/.claude/projects/{hash}/`
- 主要用于上下文恢复
- CLI 驱动，无可视化管理

**Craft Agents:**
- **JSONL 流式存储** (每行一个 JSON)
- 完整的会话元数据 (标题、标签、状态)
- **图形化会话管理**界面
- 支持会话重放
- 多工作区隔离

```jsonl
// Craft Agents 会话格式 (streaming-friendly)
{"type":"user","content":"帮我分析...","timestamp":"2026-01-22T11:38:00Z"}
{"type":"assistant","content":"好的...","timestamp":"2026-01-22T11:38:01Z"}
{"type":"tool_use","tool":"gmail:search","args":{...},"timestamp":"..."}
```

### 4. **工作区概念**

**Claude Code:**
- 以**项目目录**为中心
- 每个目录有独立的 `.claude/` 配置
- 全局设置在 `~/.claude/`

**Craft Agents:**
- 以**工作区 (Workspace)** 为中心
- 每个工作区完全隔离：
  - 独立的 Sources
  - 独立的 Sessions
  - 独立的 Skills
  - 独立的主题
  - 独立的权限规则

```
~/.craft-agent/workspaces/
├── personal/           # 个人工作区
│   ├── sources/        # Gmail, Calendar, Notion
│   ├── sessions/
│   └── skills/
└── work-acme/          # 公司工作区
    ├── sources/        # Slack, Linear, GitHub Enterprise
    ├── sessions/
    └── skills/
```

### 5. **Skills vs Slash Commands**

**Claude Code:**
- 内置 slash commands (`/commit`, `/review-pr` 等)
- 用户可以在 CLAUDE.md 中定义项目规则
- 无法创建自定义 slash commands

**Craft Agents:**
- **Skills 系统**: 用户可创建自定义技能
- Markdown 格式，易于编写和分享
- 支持 YAML frontmatter 元数据

```markdown
<!-- ~/.craft-agent/workspaces/xxx/skills/daily-standup/SKILL.md -->
---
name: Daily Standup
description: 生成每日站会汇报
icon: 📋
---

当用户调用此 Skill 时：
1. 读取 Linear 中"我的任务"
2. 读取 GitHub 中今日提交
3. 读取 Calendar 中今日会议
4. 生成结构化的站会汇报

格式要求：
- 昨日完成
- 今日计划
- 阻塞问题
```

## 🔮 技术实现差异

### 1. Agent SDK 使用方式

**Claude Code:**
```typescript
// 直接使用 Agent SDK，高度定制
import { Agent } from '@anthropic-ai/claude-agent-sdk';

const agent = new Agent({
  tools: [Read, Write, Bash, Grep, Glob, ...],
  // 内置工具深度优化
});
```

**Craft Agents:**
```typescript
// 继承 Agent SDK，增加 Sources 层
import { Agent } from '@anthropic-ai/claude-agent-sdk';

class CraftAgent extends Agent {
  private sources: SourceManager;
  
  async buildTools() {
    // 动态从 Sources 构建工具列表
    const mcpTools = await this.sources.getMcpTools();
    const apiTools = await this.sources.getApiTools();
    return [...mcpTools, ...apiTools];
  }
}
```

### 2. 工具响应处理

**Claude Code:**
- 直接返回原始响应
- 大文件使用分块读取 (offset/limit)

**Craft Agents:**
- **大响应自动摘要** (>60KB 使用 Haiku)
- 支持 `_intent` 字段聚焦摘要

```typescript
// Craft Agents 的大响应处理
if (resultSize > 60 * 1024) {
  return await summarizeWithHaiku(result, tool._intent);
}
```

### 3. 安全模型

**Claude Code:**
- 信任用户环境
- MCP 服务器继承用户权限
- 沙箱可选 (macOS Seatbelt)

**Craft Agents:**
- **零信任 MCP 环境**
- 自动过滤敏感环境变量
- 凭证永不暴露给子进程
- 所有存储 AES-256-GCM 加密

## 📊 总结：选择指南

| 如果你是... | 推荐 |
|------------|------|
| 全栈开发者，日常写代码 | **Claude Code** |
| 需要连接 Gmail/Slack/Teams | **Craft Agents** |
| 喜欢终端，效率优先 | **Claude Code** |
| 需要图形界面，团队使用 | **Craft Agents** |
| 专注单一项目开发 | **Claude Code** |
| 需要多工作区隔离 | **Craft Agents** |
| 需要自定义 AI 工作流 | **Craft Agents** |
| 需要 Web 搜索能力 | **Claude Code** |
| 需要企业级凭证管理 | **Craft Agents** |

## 🤝 它们可以互补

实际上，这两个工具可以**协同使用**：

1. **开发时用 Claude Code** - 写代码、调试、Git 操作
2. **工作流用 Craft Agents** - 整合邮件、消息、文档

Craft Agents 甚至可以把 **Claude Code 作为 MCP Source** 集成，让 GUI 用户也能享受 Claude Code 的代码能力！

---

**核心差异一句话总结:**

> **Claude Code** = 开发者的 AI 结对程序员 (CLI-first, Code-focused)
> 
> **Craft Agents** = 知识工作者的 AI 助手平台 (GUI-first, Source-centric)
