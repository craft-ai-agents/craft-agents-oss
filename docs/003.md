# Craft Agents OSS 项目深度分析报告

我已经完成了对这个项目的深入研究。以下是关键发现：

## 🎯 项目核心定位

**Craft Agents** 是一个基于 Claude Agent SDK 的桌面应用，提供文档中心化的 AI 工作流程。它将 MCP (Model Context Protocol) 集成与强大的权限系统结合，创建了一个安全、可扩展的 AI 助手平台。

## 🏗️ 技术架构亮点

### 1. **三层 Monorepo 架构**
```
apps/electron/          # Electron 桌面应用
├── main/              # Node.js 主进程
├── preload/           # 安全桥接层
└── renderer/          # React UI

packages/shared/        # 核心业务逻辑 ⭐
├── agent/             # CraftAgent (权限系统)
├── mcp/               # MCP 客户端实现
├── sources/           # 数据源管理
├── auth/              # OAuth 2.0 + PKCE
├── credentials/       # AES-256-GCM 加密
└── sessions/          # JSONL 持久化

packages/core/          # 类型定义
```

### 2. **创新的 Sources 系统**

支持三种数据源类型：

| 类型 | 用途 | 示例 |
|------|------|------|
| **MCP** | 连接 MCP 服务器 (stdio/HTTP) | GitHub, Linear, Notion |
| **API** | REST API (内置 OAuth) | Gmail, Slack, OneDrive |
| **Local** | 本地文件系统访问 | 项目目录, Git 仓库 |

### 3. **三级权限模式** (类似 Claude Code)

```typescript
'safe'      → Explore (只读，阻止所有写操作)
'ask'       → Ask to Edit (每次写操作询问，默认)
'allow-all' → Auto (自动批准所有命令)
```

**智能命令分析**：
- Bash 命令模式匹配（检测 `rm`, `git commit`, `>` 等）
- 工具级别规则（如 `Write` 总是询问）
- 源级别规则（如生产数据库强制只读）

### 4. **企业级安全机制**

**加密存储** (`~/.craft-agent/credentials.enc`):
- 算法：AES-256-GCM
- 密钥派生：基于机器 UUID + SHA-256
- 存储：OAuth tokens, API keys, 环境变量

**MCP 进程隔离**:
```typescript
// 自动过滤敏感环境变量，防止泄露给 MCP 子进程
BLOCKED_ENV_VARS = [
  'ANTHROPIC_API_KEY',
  'AWS_SECRET_ACCESS_KEY',
  'GITHUB_TOKEN',
  // ...
]
```

### 5. **OAuth 2.0 + PKCE 实现**

支持三大平台（Google, Slack, Microsoft）：

```
1. 生成 PKCE challenge
2. 启动本地 HTTP 服务器 (随机端口)
3. 打开浏览器进行授权
4. 接收回调，交换 token
5. 加密存储 access_token + refresh_token
6. 关闭浏览器和本地服务器
```

### 6. **大响应自动摘要**

当 MCP 工具返回超过 **60KB** 的数据时：
- 自动使用 **Claude Haiku** 进行摘要
- 基于工具的 `_intent` 字段聚焦关键信息
- 保留原始大小信息供追溯

## 🚀 开发工具链

### 核心技术栈
- **运行时**: Bun (20-100x 安装速度提升)
- **桌面框架**: Electron 39.2.7
- **前端**: React 18 + Vite 6
- **构建**: esbuild (主进程) + Vite (渲染进程)
- **状态管理**: Jotai (原子化状态)
- **UI**: shadcn/ui + Tailwind CSS v4
- **AI SDK**: @anthropic-ai/claude-agent-sdk 0.2.12

### 关键命令
```bash
bun run electron:dev      # 开发模式 (Hot Reload)
bun run electron:build    # 生产构建
bun test                  # 运行测试
bun run typecheck:all     # 类型检查
```

## 📂 重要文件路径

| 功能 | 路径 |
|------|------|
| Agent 核心 | `packages/shared/src/agent/index.ts` |
| MCP 客户端 | `packages/shared/src/mcp/client.ts` |
| 权限验证 | `packages/shared/src/agent/permissions.ts` |

| IPC 处理器 | `apps/electron/src/main/ipc/*.ts` |
| React 入口 | `apps/electron/src/renderer/App.tsx` |

## 💡 项目特色功能

1. **Skills 系统**: Markdown 格式的专业化 Agent 指令
2. **动态状态工作流**: 可自定义的任务状态（Todo → In Progress → Done）
3. **会话重放**: JSONL 格式支持流式读取和重放
4. **组件 Playground**: 独立的 UI 组件开发预览系统
5. **Deep Linking**: `craftagents://` 协议支持
6. **多工作区**: 完全隔离的工作区配置和会话

## 🔧 快速开始建议

```bash
# 1. 安装依赖
bun install

# 2. 配置环境变量（可选，OAuth 功能需要）
cp .env.example .env

# 3. 启动开发服务器
bun run electron:dev

# 4. 构建生产版本
bun run electron:build
```
