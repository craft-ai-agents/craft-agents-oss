## Executive Summary

Vespr is building the **universal operational orchestration platform** that applies to any team managing daily priorities and workflows.

The same loop—Daily Context → Intelligent Triage → Orchestration → Execution → Review—works for:
- **Engineering** (bugs, PRs, features)
- **Finance** (invoices, reconciliation, AP)
- **HR/Recruitment** (candidates, interviews, offers)
- **Operations** (support tickets, SLAs, escalations)
- **Marketing** (content, scheduling, campaigns)

**The Strategy:**
- **Phase 1 (Jan-May 2026):** Prove the loop with engineering teams
- **Phase 2-5 (Jun 2026 - Aug 2027):** Expand to other verticals
- **Land-and-expand:** Single company using 3-5 vertical adapters
- **Result:** 5-10x ARPU per customer, $1.76B TAM, 2-3x higher exit multiple

## The Orchestration Loop (Universal Template)

**Works for any operational team:**

```
┌──────────────────────────────────────────────────────────────┐
│                     UNIVERSAL LOOP                            │
├──────────────────────────────────────────────────────────────┤
│                                                                │
│  1. CONTEXT     → Read daily work (issues, tasks, errors)    │
│  2. TRIAGE      → AI scores by impact × urgency / complexity  │
│  3. ORCHESTRATE → Decompose, assign, sequence                │
│  4. EXECUTE     → AI handles routine work                    │
│  5. REVIEW      → Human approves results                     │
│                                                                │
│  OUTCOME: 1.5-2x more work shipped in same time              │
│                                                                │
└──────────────────────────────────────────────────────────────┘
```

### Applied to Each Vertical

**Engineering:**
- Context: GitHub issues, PRs, errors
- Triage: "What impacts users most?"
- Execute: Auto-assign, PR review, deploy
- Review: Ship or request changes

**Finance:**
- Context: Invoices, receipts, payment requests
- Triage: "What's due first? What's largest?"
- Execute: Process invoices, reconcile accounts, pay vendors
- Review: CFO approves payments

**HR/Recruitment:**
- Context: Job applicants, interview schedule, offer pipeline
- Triage: "Which candidates are best fit? When are interviews?"
- Execute: Screen resumes, schedule interviews, send offers
- Review: Hiring manager approves offers

**Operations:**
- Context: Support tickets, error logs, SLAs
- Triage: "What's critical? What's waiting longest?"
- Execute: Route tickets, suggest resolutions, escalate
- Review: Ops manager approves escalations

**Marketing:**
- Context: Content calendar, social posts, campaign ideas
- Triage: "What content performs best? What's due?"
- Execute: Draft posts, schedule social, optimize copy
- Review: Content lead approves before posting

## Risk Management

### Phase 1 Risks
- **Risk:** GitHub API rate limiting slows daily reports
  - **Mitigation:** Aggressive caching (1-hour TTL), batch API calls

- **Risk:** Triage algorithm accuracy not trusted
  - **Mitigation:** Show reasoning, allow override, collect weekly feedback

- **Risk:** Engineering leaders don't approve assignments
  - **Mitigation:** 1-click approval, clear reasoning, multiple communication channels

- **Risk:** System gets out of sync with GitHub
  - **Mitigation:** Webhook integration + hourly reconciliation

### Phase 2-5 Risks
- **Risk:** Other verticals don't adopt (land-and-expand fails)
  - **Mitigation:** Prove ROI in engineering first, get internal champions

- **Risk:** Vertical-specific features too different (product becomes unfocused)
  - **Mitigation:** Core loop is identical, only integrations differ

- **Risk:** Competitor launches multi-vertical platform
  - **Mitigation:** Speed barrier = 18+ months to replicate
